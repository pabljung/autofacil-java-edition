# Relatório do Projeto AutoFácil Java Edition

## Introdução

Este relatório documenta o desenvolvimento do sistema de locadora de veículos "AutoFácil Java Edition", um projeto educacional voltado para alunos dos semestres iniciais de Engenharia de Software e Engenharia da Computação. O sistema foi desenvolvido utilizando conceitos de programação orientada a objetos em Java, interfaces gráficas com Swing e persistência de dados em arquivos JSON.

## Objetivos do Projeto

O objetivo principal foi criar um sistema desktop simplificado para gerenciar as operações básicas de uma locadora de veículos, incluindo:

- Gerenciamento de veículos (cadastro, listagem, busca, atualização e remoção)
- Gerenciamento de clientes (cadastro, listagem e busca)
- Gerenciamento de aluguéis (registro de saída, registro de devolução e listagem)

## Tecnologias Utilizadas

### Java
Linguagem de programação principal do projeto, utilizada para implementar toda a lógica de negócio e estrutura das classes.

### Java Swing
Biblioteca para criação da interface gráfica do usuário (GUI), proporcionando uma experiência visual intuitiva para o sistema.

### Gson (Google)
Biblioteca para serialização e desserialização de objetos Java para/de formato JSON, utilizada para persistência de dados em arquivos.

## Classes Implementadas

### Classes de Domínio

#### Veiculo.java
Classe que representa um veículo na locadora, contendo:
- Atributos: placa, marca, modelo, ano, cor, precoDiaria, status
- Métodos: construtores, getters/setters, alugar(), devolver()

#### Cliente.java
Classe que representa um cliente da locadora, contendo:
- Atributos: cpf, nome, telefone, email
- Métodos: construtores, getters/setters

#### Aluguel.java
Classe que representa um aluguel, contendo:
- Atributos: idAluguel, veiculo, cliente, dataSaida, dataPrevistaDevolucao, dataDevolucaoEfetiva, valorTotal, statusAluguel
- Métodos: construtores, getters/setters, finalizarAluguel()

### Classe de Persistência

#### GerenciadorArquivos.java
Classe utilitária responsável pela persistência de dados em arquivos JSON:
- Método salvarDados(): serializa listas de objetos para JSON
- Método carregarDados(): desserializa dados JSON para objetos Java

### Classes da Interface Gráfica

#### JanelaPrincipal.java
Janela principal da aplicação que organiza todos os painéis em abas (JTabbedPane).

#### Painéis de Veículos
- PainelCadastroVeiculo.java: formulário para cadastro de novos veículos
- PainelListagemVeiculos.java: tabela para exibição dos veículos cadastrados

#### Painéis de Clientes
- PainelCadastroCliente.java: formulário para cadastro de novos clientes
- PainelListagemCliente.java: tabela para exibição dos clientes cadastrados

#### Painéis de Aluguéis
- PainelRegistroAluguel.java: formulário para registro de novos aluguéis
- PainelDevolucaoAluguel.java: formulário para registro de devoluções
- PainelListagemAlugueis.java: tabela para exibição do histórico de aluguéis

## Funcionalidades Implementadas

### Gerenciamento de Veículos
- Cadastro de veículos com validação de dados
- Listagem de veículos em formato tabular
- Controle de status (Disponível/Alugado)

### Gerenciamento de Clientes
- Cadastro de clientes com informações básicas
- Listagem de clientes cadastrados

### Gerenciamento de Aluguéis
- Registro de aluguéis com validação de cliente e veículo
- Cálculo automático do valor total baseado na duração do aluguel
- Registro de devoluções com atualização do status do veículo
- Histórico completo de aluguéis

## Pontos Fortes do Sistema

1. **Arquitetura Bem Estruturada**: Separação clara entre classes de domínio, persistência e interface gráfica.

2. **Interface Intuitiva**: Uso de abas para organizar as funcionalidades de forma lógica e acessível.

3. **Persistência Eficiente**: Utilização do Gson para serialização/desserialização automática de objetos.

4. **Validações Implementadas**: Verificação de existência de clientes e disponibilidade de veículos antes de registrar aluguéis.

5. **Feedback ao Usuário**: Mensagens de sucesso e erro utilizando JOptionPane.

## Desafios Encontrados

1. **Integração entre Componentes**: Conectar a interface gráfica com a lógica de negócio e persistência de dados.

2. **Gerenciamento de Estado**: Manter a consistência dos dados entre diferentes painéis da aplicação.

3. **Tratamento de Exceções**: Implementar tratamento adequado para erros de I/O e parsing de dados.

## Melhorias Futuras

1. **Validação de Dados**: Implementar validações mais robustas para CPF, datas e outros campos.

2. **Busca Avançada**: Adicionar funcionalidades de busca por diferentes critérios.

3. **Relatórios**: Gerar relatórios de faturamento e estatísticas de uso.

4. **Backup de Dados**: Implementar sistema de backup automático dos arquivos JSON.

## Conclusão

O projeto AutoFácil Java Edition foi desenvolvido com sucesso, atendendo a todos os requisitos especificados. O sistema demonstra a aplicação prática de conceitos fundamentais de programação orientada a objetos, desenvolvimento de interfaces gráficas e persistência de dados. A estrutura modular e bem organizada facilita futuras manutenções e expansões do sistema.

O projeto serve como uma excelente base para o aprendizado de desenvolvimento de aplicações desktop em Java, proporcionando aos estudantes uma experiência prática com tecnologias amplamente utilizadas na indústria.
